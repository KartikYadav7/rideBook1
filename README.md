# 🚖 RideBook – Full Stack Cab, Package & Reservation Booking App\n\nRideBook is a full-stack web app for booking **cabs**, **sending packages**, and **reserving drivers**. It includes real-time bookings, Stripe payments, driver/cab assignment, email notifications, and an admin panel for managing users, bookings, and resources.\n\n---\n\n## 🖼️ Screenshots\n\n### ✅ Booking Flow\n![Booking Page](screenshots/booking-page.png)\n\n### ✅ Payment Success\n![Payment Success](screenshots/payment-success.png)\n\n### ✅ Admin Dashboard\n![Admin Dashboard](screenshots/admin-dashboard.png)\n\n> 📂 Place your screenshots in the `/screenshots/` folder.\n\n---\n\n## 🚀 Features\n\n- ✅ Ride, Package, Reservation Bookings\n- ✅ Stripe Payment Integration\n- ✅ Driver & Cab Auto-assignment\n- ✅ Booking Status Tracking\n- ✅ Email Notifications (User + Driver)\n- ✅ Booking Cancellation Support\n- ✅ Admin Panel for managing bookings/resources\n- ✅ JWT Auth for users and admin\n\n---\n\n## 🏗️ Tech Stack\n\n### Frontend\n- React + Redux\n- TailwindCSS / Bootstrap\n- Axios for API requests\n\n### Backend\n- Node.js + Express\n- MongoDB + Mongoose\n- Stripe API for payments\n- Nodemailer for email templates\n\n---\n\n## ⚙️ Setup Instructions\n\n### 1. Clone the repo\n\n```bash\ngit clone https://github.com/yourusername/ridebook.git\ncd ridebook\n```\n\n### 2. Install dependencies\n\n```bash\nnpm install\ncd client\nnpm install\n```\n\n### 3. Configure environment\n\n#### Backend `.env`\n\n```env\nPORT=5000\nMONGO_URI=mongodb://localhost:27017/ridebook\nSTRIPE_SECRET_KEY=sk_test_...\nFRONTEND_URL=http://localhost:3000\nEMAIL_USER=your_gmail@gmail.com\nEMAIL_PASS=your_gmail_app_password\nJWT_SECRET=your_jwt_secret_key\n```\n\n#### Frontend `client/.env`\n\n```env\nREACT_APP_API=http://localhost:5000\nREACT_APP_STRIPE_PUB_KEY=pk_test_...\n```\n\n---\n\n## ▶️ Run the App\n\n### Backend\n```bash\nnpm run dev\n```\n\n### Frontend\n```bash\ncd client\nnpm start\n```\n\n---\n\n## 📦 API Overview\n\n### Auth Routes\n\n| Method | Endpoint | Description       |\n|--------|----------|-------------------|\n| POST   | `/api/auth/signup` | Register new user |\n| POST   | `/api/auth/login`  | Login with JWT    |\n\n### Booking Routes\n\n| Method | Endpoint | Description                |\n|--------|----------|----------------------------|\n| POST   | `/api/bookings/:type` | Create booking (ride/package/reservation) |\n| PATCH  | `/api/bookings/cancel/:type/:bookingId` | Cancel booking     |\n| GET    | `/api/bookings/user/:userId` | Get all user bookings |\n\n### Payment Routes\n\n| Method | Endpoint | Description             |\n|--------|----------|-------------------------|\n| POST   | `/api/payment/create-checkout-session` | Start Stripe session |\n| GET    | `/api/payment/status?session_id=...`   | Check payment status |\n\n---\n\n## 📁 Project Structure\n\n```\nridebook/\n├── client/               # React frontend\n├── controllers/          # Express controllers\n├── models/               # Mongoose schemas\n├── routes/               # Express routes\n├── utils/                # Helper functions (emails, price calc, etc.)\n├── templates/            # HTML email templates\n├── screenshots/          # App screenshots\n├── server.js             # Main backend entry\n└── .env                  # Environment config\n```\n\n---\n\n## 📬 Emails\n\nEmails are sent using **Nodemailer** with HTML templates for:\n\n- Booking Confirmation (User + Driver)\n- Booking Cancellation (User + Driver)\n\n---\n\n## 🛠 Admin Panel\n\nAccess `/admin` route to manage:\n\n- Bookings\n- Drivers\n- Cabs\n- Users\n\n> 🔒 Admin panel protected via JWT auth.\n\n---\n\n## 🧪 Sample Test Accounts\n\n| Role  | Email               | Password    |\n|-------|---------------------|-------------|\n| User  | testuser@mail.com   | 123456      |\n| Admin | admin@ridebook.com  | admin123    |\n\n---\n\n## 🤝 Contributing\n\nPull requests and feedback are welcome!\n\n---\n\n## 📄 License\n\nMIT License © 2024 RideBook Team
